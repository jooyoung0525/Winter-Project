2.Bagging and Pasting¶

Bagging: 샘플링 할 때 replace가 있는것(같은 data sample 중복으로 사용 )

Pasting: 샘플링 할 때 replace가 없는것(같은 data sample 중복으로 사용하지 않음 )

앙상블이 original 학습 set에서 학습된 개별 예측기와 bias는 비슷하지만 더 낮은 variance를 가짐.

bagging과 pasting 이 많이 사용되는 이유 중 하나는 확장성이 좋기 때문이다. (예를들어 예측기는 서로 다른 CPU코어 또는 다른 서버를 통해 병렬로 학습 시킬 수 있고 예측을 병렬로 수행 할 수 있음.)

2.1 Bagging and Pasting in Scikit-Learn¶

scikit-learn 은 BaggingClassifier클래스를 사용할 수 있는 간단한 API를 제공.

from matplotlib.colors import ListedColormap

def plot_decision_boundary(clf, X, y, axes=[-1.5, 2.5, -1, 1.5], alpha=0.5, contour=True):
    x1s = np.linspace(axes[0], axes[1], 100)
    x2s = np.linspace(axes[2], axes[3], 100)
    x1, x2 = np.meshgrid(x1s, x2s)
    X_new = np.c_[x1.ravel(), x2.ravel()]
    y_pred = clf.predict(X_new).reshape(x1.shape)
    custom_cmap = ListedColormap(['#fafab0','#9898ff','#a0faa0'])
    plt.contourf(x1, x2, y_pred, alpha=0.3, cmap=custom_cmap, linewidth=10)
    if contour:
        custom_cmap2 = ListedColormap(['#7d7d58','#4c4c7f','#507d50'])
        plt.contour(x1, x2, y_pred, cmap=custom_cmap2, alpha=0.8)
    plt.plot(X[:, 0][y==0], X[:, 1][y==0], "yo", alpha=alpha)
    plt.plot(X[:, 0][y==1], X[:, 1][y==1], "bs", alpha=alpha)
    plt.axis(axes)
    plt.xlabel(r"$x_1$", fontsize=18)
    plt.ylabel(r"$x_2$", fontsize=18, rotation=0)

위 그림은 하나의 의사결정 트리의 의사결정 경계와 500개의 트리로 앙상블의 결정경계와 비교모습이다. (data는 moons dataset이용)

앙상블의 예측은 단일 의사결정 트리의 예측보다 훨씬 일반적임을 알 수 있다. 앙상블은 비슷한 bias를 가지지만 variance는 더 적다. bagging은 각 예측기가 학습된 subset에서 조금 더 다양성을 지닌다. 따라서 pasting 보다 약간 높은 bias를 가지게 된다. 하지만 subset에서 다양성을 지니는 것은 예측기들의 상관관계가 낮아지기 때문에 앙상블의 variance가 줄어든다. 이러한 점은 bagging이 더 좋은 모델을 만들어내고 선호하는 이유이다.

bagging과 pasting을 평가하기 위해 cross-validation을 이용할 수 있고, 더 잘 작동하는 것을 선택할 수 있다.

2.2 Out-of-Bag Evaluation¶

bagging을 사용하면 일부 인스턴스들은 여러번 샘플링 될 수 있지만, 다른 인스턴스들은 전혀 샘플링 되지 않을 수 있다.

기본적으로 Baggingclassifier은 m개의 학습 인스턴스를 replacement(bootstrap = True)로 샘플링된다. (m은 학습set의 크기)

만약 학습 인스턴스의 약 63%만 각 예측기에 대해 평균 샘플링 되었다면 나머지 샘플링 되지 않은 학습 인스턴스의 37%는 out-of-bag(oob)인스턴스라고 한다. 예측기는 학습 동안에 oob인스턴스를 보지 못하기 때문에 oob인스턴스들은 평가에 이용될 수 있고 별도의 유효성 검증 set 또는 cross-validation이 필요 없다. 각 예측기의 oob평가를 평균하여 앙상블 자체를 평가 할 수 있다.

3. Random Patches and Random Subspaces¶

BaggingClassifier클래스는 feature 샘플링도 지원. 두개의 하이퍼 파라미터 max_features와 bootstrap_features에 의해 제어된다. (image와 같은 고차원 inputs값에 대해 다룰 때 유용)

Random Patches method : 학습 인스턴스와 features 모두 샘플링 하는 것.

Random Subspaces method : 모든 학습 인스턴스들은 유지하고 샘플링 feature(bootstrap_feature = True, and/or max features는 1보다 작게)로 하는 방식

샘플링feature는 낮은 variance를 위해 좀 더 bias를 가지게 되어 예측기의 다양성을 확보 할 수 있다.

5.Boosting¶

Boosting은 여러 weak learners를 결합하여 strong learner로 되기 위한 앙상블 방법. 대부분의 boosting방법으로는 예측기들의 각각 predecessor을 수정하면서 순차적으로 예측기를 학습시키는것.

가장널리 사용하는 방법으로는 AdaBoost와 Gradient Boosting이 있다.

Adaboost¶

under-fitted된 predecessor의 학습 인스턴스에 좀 더 주의를 기울여 새로운 예측기의 predecessor을 수정하는 방법이 있다. (Adaboost에서 사용하는 기법.)

Adaboost분류기 작성하려면 의사결정트리와 같은 첫 번째 기본 분류기가 학습되고 학습set에 대한 예측을 수행하는 데 사용된다. 두번째 분류기는 업데이트 된 weight를 이용해 학습되고 다시 학습 set에 대한 예측을 수행하고 weight는 업데이트 된다 이를 계속 반복한다.

def plot_predictions(regressors, X, y, axes, label=None, style="r-", data_style="b.", data_label=None):
    x1 = np.linspace(axes[0], axes[1], 500)
    y_pred = sum(regressor.predict(x1.reshape(-1, 1)) for regressor in regressors)
    plt.plot(X[:, 0], y, data_style, label=data_label)
    plt.plot(x1, y_pred, style, linewidth=2, label=label)
    if label or data_label:
        plt.legend(loc="upper center", fontsize=16)

Gradient Boosting with Early stopping¶

최적의 트리수를 찾으려면 early stopping(4장에서 다룸)을 사용할 수 있다. 이는 staged_predict()메소드를 사용해 구현 가능하다.

아래의 코드는 120개의 트리를 가진 GBRT앙상블을 훈련시킨 다음 학습의 각 단계에서 valisation error을 측정하고 최적의 트리수를 찾아내어 마지막으로 최적의 트리수를 사용하여 다른 GBRT앙상블을 학습시킨다.

In [43]:
import numpy as np
from sklearn.model_selection import train_test_split
from sklearn.metrics import mean_squared_error

X_train, X_val, y_train, y_val = train_test_split(X, y, random_state=49)

gbrt = GradientBoostingRegressor(max_depth=2, n_estimators=120, random_state=42)
gbrt.fit(X_train, y_train)

errors = [mean_squared_error(y_val, y_pred)
          for y_pred in gbrt.staged_predict(X_val)] # early stopping 구현
bst_n_estimators = np.argmin(errors)

gbrt_best = GradientBoostingRegressor(max_depth=2,n_estimators=bst_n_estimators, random_state=42)
gbrt_best.fit(X_train, y_train)

Validation errors는 왼쪽, 가장 좋은 모델의 예측은 오른쪽이다.

학습을 실제로 일찍 중단시킴으로써 early stopping을 구현할 수 있다.

아래 코드는 validation error가 연속적으로 5회 반복되지 않으면 학습을 중지한다.

In [46]:
gbrt = GradientBoostingRegressor(max_depth=2, warm_start=True, random_state=42)
# warm_start=True이면 fit()메소드가 호출될 때, scikit-learn은 기존 트리를 유지하면서 점진적 학습을 허용한다.

min_val_error = float("inf")
error_going_up = 0
for n_estimators in range(1, 120):
    gbrt.n_estimators = n_estimators
    gbrt.fit(X_train, y_train)
    y_pred = gbrt.predict(X_val)
    val_error = mean_squared_error(y_val, y_pred)
    if val_error < min_val_error:
        min_val_error = val_error
        error_going_up = 0
    else:
        error_going_up += 1
        if error_going_up == 5: # validation error가 5회 이상 반복되지 않으면 학습을 중지한다.
            break  # early stopping

Stacking¶

그림 7-12에서 새로운 인스턴스에서 회귀 작업을 수행하는 앙상블을 보여준다. 하위 3개의 예측은 각각 다른 값을 예측한다. 그리고 최종 예측기(blender)에서는 이러한 예측을 입력값으로 사용하여 최종 예측을 수행한다.

blender를 학습 하기 위한 일반적인 방법은 hold-out set을 사용하는 것.

<stacking의 동작> 학습 set을 두 개의 subset으로 나눔.

  1. 첫번째 subset은 첫번째 layer의 예측기를 학습하는데 사용.
  2. 첫번째 layer의 예측기는 두번째 set에 대한 예측을 하는데 사용
  3. hold-out(보류된) set의 각 인스턴스에 대해 세가지 예측된 값을 feature로 사용하여 새로운 학습 set을 만들 수 있고 목표값을 유지시킨다. blender는 새로운 학습 set에 대해 학습되고, 첫 번째 layer의 예측에서 주어진 targer value를 예측하기 위해 학습한다.

blender 학습의 요령은 다음과 같다. 학습set을 세개의 subset으로 분리한다.

  1. 첫 번째 subset은 layer1를 학습하기 위해 사용된다.
  2. 두 번째 subset은 layer2를 학습하는 데 사용되는 학습 set을 만드는 데 사용된다.
  3. 세 번째 subset은 layer3를 학습시키는 학습 set을 만드는 데 사용된다. 이러한 작업이 완료되면 그림 7-15와 같이 각 계층을 순차적으로 통과하여 새로운 인스턴스에 대해 예측을 할 수 있다.
